# Phase 03.1-01 Summary

## Scope

Ізолювати interactive board onboarding від routine `gateway-main` silent control-plane session і прибрати допустимість `NO_REPLY` як "успішного" кроку.

## Implemented

- У `backend/app/services/openclaw/internal/session_keys.py` додано `gateway_onboarding_session_key(gateway_id, board_id)`.
- У `backend/app/services/openclaw/onboarding_service.py` onboarding start dispatch переведено з `GatewayAgentIdentity.session_key(gateway)` на board-scoped onboarding session key.
- У `backend/app/api/board_onboarding.py` onboarding prompt винесено в `_build_onboarding_prompt()` і доповнено явним guard:
  - `NO_REPLY is invalid for this workflow.`
  - агент має або надіслати next question, або completion payload.
- Додано backend regression tests на:
  - dedicated onboarding session selection;
  - prompt-level `NO_REPLY` guard;
  - deterministic onboarding session key format.

## Verification

## Outcome

Interactive onboarding більше не стартує в routine `agent:mc-gateway-...:main` session family без окремої board-scoped isolation layer, а onboarding contract більше не допускає `NO_REPLY` як валідний happy-path.
